Perfecto ya todo esta esta ok, continuamos con el desarrollo, ahora hay que crear un microservicio 
que permite sincronizar datos entre base de datos postgresql y oracle 11gr2, las cuales llamaremos de la 
siguiente manera

oracle 11gr2 => bd_remota
postgresql => bd_local

la sincronización se realizara bajo la siguiente estructura

las siguiantes tablas:
maestro
centros_costo
rrhh_asigna_trjt_reloj

solo se sicnronizaran en una sola direcion, de bd_remota a bd_local, 

mientras que la siguiente tabla
asistencia_ht580

solo se sincronizara tambien en una sola dirección, de bd_local a bd_remota

de esa manera en la bd_local se traeran los datos remotos, teniendo como una copia del mismo, y es el registro
de asistencia que es la que se sube a la bd_remota.

la sincronizacion se realizara cada 5 min, disparando dos hilos una para sincronizacion de bd_remota a bd_local, 
y la otra de bd_local a bd_remota.

una vez que ambos hilos terminen (debe haber algun tipo de sincronizador de tareas) se lanza nuevamente 
el proceso de sincronía despues de 5 min, osea que el proceso no es que cora cada 5 min, sino que se dispara
cada 5 min una vez que termine de sincronizar, asi se evitan que hayan tareas que se junten por demora en la
sincronización o saturación por lo mismo por fallas en la comunicación

si por algun motivo hay una falla en l comunidación se terminan las tareas, se esperan 30 seg se reintenta una vez mas
y si continua el error, se termina el proceso, y luego es lanza nuevamente el proceso apasados los 5 min.
