HU-RRHH-CONF-PAR-001 - Configuración de Frecuencia y Calendarios de Pago Multipaís - v1.03
1. Título
 Configura frecuencia de pago (semanal, quincenal, mensual) y fechas de corte. 
2. Descripción
 Como usuario administrador del módulo de Recursos Humanos de Restaurant.pe, quiero configurar la frecuencia de pago  y las fechas de corte de nómina, para garantizar que los procesos de cálculo y pago de sueldos se ejecuten correctamente según la normativa y costumbres laborales de cada país..[a][b]
3. Proceso
 Proceso: Configuración General
Subproceso: Parámetros del Módulo
Módulo: Recursos Humanos (RR.HH.)
4. Narrativa de Usuario
 Quiero: Configurar la frecuencia de pago y las fechas de corte.
Para: Asegurar que el proceso de nómina se ejecute en los periodos correctos y según las leyes locales.
De manera que: El sistema procese automáticamente las remuneraciones, evitando errores en los cálculos y cumpliendo las obligaciones laborales de cada país.
5. Regla de Navegabilidad
 Ruta: Menú Principal > RR.HH. > Configuración > Parámetros del Módulo > Frecuencia y Calendarios de Pago
6. Criterios de Aceptación
 El sistema debe permitir:
* Configurar la frecuencia de pago por sede o grupo de empleados (semanal, quincenal, mensual u otra personalizada).

* Definir las fechas de inicio y fin de periodo, así como la fecha de pago correspondiente.

* Establecer fechas de corte para el cierre de horas trabajadas, comisiones o variables antes del cálculo de nómina.

* Asignar frecuencias diferentes por país, respetando las normas laborales locales.

* Mostrar alertas o bloqueos cuando las fechas de corte se superpongan o interfieran con otro proceso de nómina.

* Permitir copiar configuraciones de un país o sede a otra.

* Validar que los empleados estén correctamente asociados a una frecuencia de pago antes de ejecutar la nómina.

* Registrar toda modificación, creación o eliminación de configuraciones en el log de auditoría contable (usuario, fecha, hora, acción).

7. Campos del Formulario / Columnas de Vista Tabla
 Campos principales:

   * Frecuencia de Pago (Semanal / Quincenal / Mensual / Personalizada)

   * Fecha de Inicio del Periodo

   * Fecha de Fin del Periodo

   * Fecha de Pago

   * Fecha de Corte

   * Estado (Activo / Inactivo)

Filtros disponibles:

      * Estado

      * Frecuencia de Pago

Acciones disponibles:
         * Crear Nueva Configuración

         * Editar

         * Copiar Configuración

         * Eliminar

         * Exportar (Excel / PDF)

         * Ver Historial de Cambios

8. Detalle de Integración:


 Integración con Nómina: para que los cálculos de sueldos, horas extras y descuentos se ejecuten según las fechas configuradas.

Integración con Contabilidad: para programar los pagos de planillas y provisiones según los periodos definidos.

Integración con RR.HH. - Empleados: para validar la frecuencia de pago asignada a cada trabajador.

Integración con Módulo de Horarios y Asistencias para que el sistema realice el cálculo de las horas programadas según horario y marcaciones.


9. Consideraciones Técnicas
 El sistema debe:.

            * Permitir la configuración de periodos variables (ej. 28 días, 30 días, 31 días).

            * Validar automáticamente los solapamientos de fechas de corte y pago.

            * Registrar los cambios de configuración en la tabla de auditoría.

            * Mostrar un resumen gráfico del calendario de pagos por sede o país.

            * Estar preparado para futuras integraciones con sistemas externos de gestión de nómina.

10. Comentarios
 Esta funcionalidad permite la gestión multipaís de nómina, adaptándose a distintas legislaciones laborales (por ejemplo, SUNAFIL en Perú, DIAN en Colombia, SII en Chile).
Se recomienda incorporar alertas automáticas cuando se acerque una fecha de corte o pago pendiente.
También puede integrarse con un módulo de planificación financiera, permitiendo anticipar el flujo de fondos requerido para el pago de sueldos.
