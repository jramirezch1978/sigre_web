HU-ALM-CON-ART-001 - Consulta de Información Completa del Producto e Imagen Referencial - v1.0
1. Título
 Consulta de toda la información del producto registrada, así como imagen referencial
2. Descripción
 Como usuario del sistema Restaurant.pe, quiero consultar toda la información registrada de un producto, incluyendo sus datos generales, clasificaciones, configuraciones contables, parámetros de inventario y su imagen referencial, para verificar su registro y asegurar la correcta gestión de artículos en el sistema.
3. Proceso
 Proceso: Consultas
Subproceso: Consulta de Artículo
Módulo: Almacén
4. Narrativa de Usuario
 Quiero: Acceder a la ficha completa de un producto con toda su información registrada en el sistema.
Para: Validar sus datos técnicos, contables, de inventario y comerciales.
De manera que: Se garantice la integridad de la información del artículo y se eviten errores en la gestión de compras, ventas y control de stock.
5. Regla de Navegabilidad
 Ruta: Menú Principal > Almacén > Consultas > Artículos > Consulta de Información de Producto
6. Criterios de Aceptación
 El sistema debe permitir:
* Consultar cualquier producto registrado mediante código, descripción, código de barras o categoría.

* Mostrar toda la información registrada del producto, incluyendo:

   * Datos generales (código, descripción, unidad de medida, estado, tipo).

   * Clasificación (categoría, subcategoría, clase de artículo).

   * Datos contables (cuentas contables asociadas, naturaleza del artículo, impuestos aplicables).

   * Datos de inventario (stock actual, punto de reposición, almacén principal).

   * Datos comerciales (proveedor habitual, precio de compra, precio de venta).

      * Mostrar la imagen referencial del producto si se encuentra registrada.

      * Permitir descargar o visualizar en pantalla la ficha del producto.

      * Incluir la opción de editar información (solo para usuarios con permisos asignados).

      * Permitir consultar el historial de modificaciones del producto (usuario, fecha, cambio realizado).

      * Permitir exportar la información en formato Excel o PDF.

      * Mostrar la razón social y sucursal asociada al producto (modo informativo).

      * Registrar toda acción de consulta o exportación en el log de auditoría contable y operativa.

7. Campos del Reporte / Columnas de Vista Tabla
 Campos principales:
         * Código del Producto dependerá de cada empresa, se recomienda priorizar los SKU por: Tipoprodutco-Categoría-Correlativo Ej.: INS-BEB-001 (Insumos / Bebidas / gaseosa personal)

         * Descripción del Producto

         * Código de Barras. Tipo: EAN-13

         * Unidad de Medida

         * Categoría / Subcategoría

         * Clase de Artículo

         * Naturaleza Contable

         * Almacén Principal

         * Stock Actual

         * Punto de Reposición

         * Precio de Compra

         * Precio de Venta

         * Proveedor Habitual

         * Impuestos Aplicables

         * Cuenta Contable de Inventario

         * Cuenta Contable de Ventas

         * Imagen Referencial

         * Estado del Artículo (Activo / Inactivo)

         * Fecha de Creación / Última Modificación

         * Usuario Responsable

Filtros disponibles:
            * Código o Descripción del Producto

            * Categoría / Subcategoría

            * Almacén

            * Estado del Producto

            * Proveedor

            * Sucursal

Acciones disponibles:
               * Ver Ficha Completa

               * Ver Imagen

               * Editar Información (según permisos)

               * Exportar (Excel / PDF)

               * Ver Historial de Modificaciones

               * Consultar Log de Auditoría

8. Detalle de Integración (opcional)
 Integración con Catálogo de Artículos: para obtener la información estructurada del producto.
Integración con Almacén: para mostrar el stock y almacenes asociados.
Integración con Compras y Ventas: para mostrar proveedores y precios relacionados.
Integración con Contabilidad: para validar cuentas contables e impuestos configurados.
Integración con Auditoría: para registrar todas las consultas y modificaciones realizadas.
9. Consideraciones Técnicas
 El sistema debe:
                  * Permitir la visualización de imágenes en formato JPG, PNG o WEBP, optimizadas para carga rápida.

                  * Garantizar la consistencia de la información entre módulos (Almacén, Compras, Contabilidad).

                  * Mostrar los datos en tiempo real o según la última actualización registrada.

                  * Permitir niveles de acceso configurables, restringiendo la edición a usuarios autorizados.

                  * Implementar una búsqueda dinámica con autocompletado.

                  * Registrar toda acción de consulta, edición o exportación en el log de auditoría contable y operativa.

                  * Soportar visualización multipaís con conversión automática de moneda para precios.[a][b]

10. Comentarios
 Esta funcionalidad es clave para el control maestro de artículos y la trazabilidad de la información.
Permite a los usuarios validar la configuración contable, logística y comercial de cada producto desde un único punto.
La imagen referencial facilita la identificación visual de los productos, reduciendo errores de selección en compras, ventas y almacenamiento.
Se recomienda incluir un indicador visual del estado del producto (Activo/Inactivo) y la posibilidad de mostrar la fecha del último movimiento de inventario.
También puede integrarse con reportes de auditoría interna para monitorear modificaciones críticas en los registros de productos.
